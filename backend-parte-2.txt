Fase 4: autenticación
Implementar autenticación obligatoria.

El usuario debe poder:
- registrarse con username
- iniciar sesión con Google
- iniciar sesión con Phantom Wallet

Google:
- validar token de Google en backend
- crear usuario si no existe
- asociar cuenta Google
- generar JWT propio

Phantom:
- generar nonce/challenge desde backend
- usuario firma mensaje con Phantom
- backend verifica firma
- crear usuario si no existe
- asociar wallet
- marcar nonce como usado
- generar JWT propio

JWT:
- debe incluir userId, username y auth provider principal
- no confiar en userId enviado por cliente
- todos los sockets deben validarse usando el JWT

Fase 6: lobby y salas
Agregar sistema de salas encima del juego actual.

Reglas:
- Las salas se crean bajo demanda.
- No hay salas fijas.
- No hay admin de sala en MVP.
- Todos los jugadores autenticados pueden jugar.
- Se mezclan jugadores normales y jugadores elegibles para recompensa.
- minPlayers = 2
- maxPlayers = 8
- countdown = 120 segundos cuando hay mínimo 2 jugadores
- si la sala se llena, countdown baja a 10 segundos
- si baja de 2 jugadores antes de iniciar, se cancela countdown
- cuando inicia la partida, se conecta esa sala con la lógica actual del juego

Estados de sala:
- waiting
- countdown
- in_game
- finished

Eventos sugeridos:
Cliente → servidor:
- lobby:listRooms
- lobby:quickPlay
- lobby:createRoom
- lobby:joinRoom
- lobby:leaveRoom
- room:getState
- game:input
- game:shoot
- game:dash

Servidor → cliente:
- lobby:roomsUpdated
- room:joined
- room:left
- room:stateUpdated
- room:countdownStarted
- room:countdownCancelled
- room:countdownUpdated
- game:started
- game:stateUpdated
- game:ended
- game:error

Si los eventos actuales del juego tienen otros nombres, mantenerlos si es posible y solo agregar los necesarios para lobby/salas.
