PROMPT PARA FRONTEND

Quiero crear el frontend de un juego 2D estilo tanques Atari/top-down.

Tecnologías obligatorias:

* Vite
* TypeScript
* Phaser 3
* Socket.IO Client
* El frontend NO debe ser la verdad del juego
* El frontend solo captura controles, envía inputs al backend y dibuja el estado recibido

Objetivo inicial:

Crear un cliente web que se conecte por WebSocket a un backend NestJS con Socket.IO, envíe inputs del jugador y dibuje el estado autoritativo recibido desde el backend.

Comandos esperados:

npm create vite@latest tank-client -- --template vanilla-ts
cd tank-client
npm i phaser socket.io-client

Estructura deseada:

src/
main.ts
game/
TankGame.ts
scenes/
BootScene.ts
GameScene.ts
network/
socket.ts
types/
game-state.types.ts
input.types.ts

Controles:

* W = moverse hacia arriba
* A = moverse hacia la izquierda
* S = moverse hacia abajo
* D = moverse hacia la derecha
* Mouse = dirección del cañón / aimAngle
* Click izquierdo = disparar

El frontend debe calcular el input así:

{
moveX: number;
moveY: number;
aimAngle: number;
shoot: boolean;
}

Reglas de movimiento:

Si se presiona W:

moveY -= 1

Si se presiona S:

moveY += 1

Si se presiona A:

moveX -= 1

Si se presiona D:

moveX += 1

Si se mueve en diagonal, se puede enviar así, pero el backend será quien normalice y valide.

Dirección del mouse:

Usar Phaser.Math.Angle.Between para calcular el ángulo entre el tanque del jugador y la posición del mouse.

Ejemplo:

const aimAngle = Phaser.Math.Angle.Between(
myPlayer.x,
myPlayer.y,
pointer.worldX,
pointer.worldY
);

El frontend debe enviar inputs al backend varias veces por segundo, por ejemplo 30 veces por segundo o dentro del update de Phaser con control de tiempo.

Eventos WebSocket:

Frontend -> Backend:

* joinGame
* playerInput
* startGame

Backend -> Frontend:

* gameJoined
* gameStarted
* gameState
* playerDisconnected

Funcionamiento esperado:

1. Al iniciar la escena, el cliente se conecta al backend.
2. Envía joinGame.
3. Recibe gameJoined con su playerId.
4. Renderiza el mapa recibido desde gameState.
5. Captura teclado y mouse.
6. Envía playerInput al backend.
7. Recibe gameState desde el backend.
8. Dibuja jugadores, balas y obstáculos según el estado recibido.
9. Permitir iniciar la partida por ahora con una tecla, por ejemplo ENTER, enviando startGame al backend.

Dibujo del mapa:

El backend enviará obstáculos con estos tipos:

* bush
* wood
* rock
* steel
* mirror

Dibujarlos de forma sencilla usando rectángulos de Phaser, sin assets por ahora.

Colores sugeridos:

* bush: verde
* wood: marrón
* rock: gris
* steel: azul oscuro o gris metálico
* mirror: cyan o azul claro

Cada obstáculo se dibuja según:

{
id: string;
type: 'bush' | 'wood' | 'rock' | 'steel' | 'mirror';
x: number;
y: number;
width: number;
height: number;
hp: number;
destructible: boolean;
}

Cuando el backend elimine un obstáculo del gameState, el frontend también debe dejar de dibujarlo.

Dibujo de jugadores:

Cada jugador debe dibujarse como un cuerpo de tanque y una línea o cañón apuntando hacia aimAngle.

Por ahora se puede usar:

* círculo o rectángulo para el cuerpo
* línea para el cañón
* texto pequeño con hp encima

El jugador propio puede tener un color diferente al resto.

Dibujo de balas:

Cada bala debe dibujarse como un círculo pequeño.

El frontend no calcula la vida real de la bala ni las colisiones. Solo dibuja las balas que el backend mande.

Importante:

El frontend NO debe decidir:

* daño
* muerte
* destrucción de obstáculos
* colisión real
* posición real
* ganador
* resultado

El frontend solo puede hacer:

* capturar teclado
* capturar mouse
* enviar input
* dibujar estado
* hacer interpolación visual más adelante, pero no por ahora

Tipos esperados:

PlayerInput:

{
moveX: number;
moveY: number;
aimAngle: number;

}

GameState:

{

{

{
id: string;

x: number;
y: number;
width: number;

hp: number;
destructible: boolean;
}

Resultado esperado:

Genera el código completo del frontend con Phaser 3, TypeScript y Socket.IO Client.

El cliente debe poder conectarse al backend, enviar inputs, iniciar la partida con ENTER y dibujar el estado recibido.

No usar React por ahora. Solo Angular 22 + TypeScript + Phaser.
