PROMPT PARA BACKEND

Quiero crear el backend autoritativo de un juego multiplayer 2D estilo tanques Atari/top-down.

Tecnologías obligatorias:

* NestJS
* TypeScript
* Socket.IO usando @nestjs/websockets
* Estado de partida en memoria RAM, sin base de datos por ahora
* Arquitectura modular y limpia
* No usar PostgreSQL todavía
* No usar Redis todavía
* El backend debe ser la verdad absoluta del juego

Objetivo inicial:

Crear un backend que permita conectar jugadores por WebSocket, crear una partida en memoria, iniciar la partida manualmente desde el backend o mediante un evento WebSocket llamado startGame, recibir inputs de los jugadores, actualizar el estado del juego en un game loop y enviar el estado actualizado a todos los clientes.

Reglas importantes:

El cliente NO debe enviar su posición real.
El cliente solo debe enviar inputs:

{
moveX: number;
moveY: number;
aimAngle: number;
shoot: boolean;
}

El backend debe validar esos inputs y calcular:

* posición real del jugador
* dirección del cañón
* creación de balas
* movimiento de balas
* colisiones
* daño
* destrucción de objetos del mapa
* rebote/reflejo de balas en espejos
* vida de jugadores, aunque al inicio puede ser básico

Eventos WebSocket requeridos:

Cliente -> Backend:

* joinGame
* playerInput
* startGame

Backend -> Cliente:

* gameJoined
* gameStarted
* gameState
* playerDisconnected

Funcionamiento esperado:

1. Cuando un cliente se conecta, puede enviar joinGame.
2. El backend registra al jugador en memoria.
3. El backend crea un jugador con id, x, y, hp, aimAngle y cooldown de disparo.
4. El backend tendrá una función que crea el mapa.
5. Por ahora el mapa debe ser predefinido, no aleatorio.
6. Más adelante podría usarse una semilla para generar mapas, pero por ahora solo dejar preparada la idea.
7. El evento startGame inicia el game loop.
8. El game loop corre a 30 ticks por segundo.
9. En cada tick el backend:

   * procesa inputs
   * mueve jugadores
   * mueve balas
   * detecta colisiones
   * aplica daño
   * actualiza el mapa
   * envía gameState a todos los clientes

Mapa inicial:

Crear un mapa sencillo predefinido con bloques o tiles. El mapa debe tener obstáculos de diferentes tipos:

* arbusto
* madera
* roca
* acero
* espejo

Cada objeto del mapa debe tener:

{
id: string;
type: 'bush' | 'wood' | 'rock' | 'steel' | 'mirror';
x: number;
y: number;
width: number;
height: number;
hp: number;
destructible: boolean;
}

Reglas de daño de obstáculos:

* arbusto: se destruye con 1 disparo
* madera: se destruye con 2 disparos
* roca: se destruye con 3 disparos
* acero: prácticamente no se daña, debe tener destructible false o hp muy alto
* espejo: no se destruye por ahora, pero refleja la bala

Regla especial del espejo:

Cuando una bala colisione con un espejo, la bala debe reflejarse en la dirección opuesta literal.

Si la bala venía con:

dirX = 1
dirY = 0

Debe quedar:

dirX = -1
dirY = 0

Si venía con:

dirX = 0.7
dirY = 0.7

Debe quedar:

dirX = -0.7
dirY = -0.7

Es decir:

bullet.dirX *= -1;
bullet.dirY *= -1;

Después del reflejo, la bala debe continuar viva y moverse en la nueva dirección.

Reglas de balas:

Cada bala debe tener:

{
id: string;
ownerId: string;
x: number;
y: number;
dirX: number;
dirY: number;
speed: number;
damage: number;
radius: number;
lifeTime: number;
}

Cuando el jugador dispara:

* El cliente solo manda shoot: true.
* El backend valida cooldown.
* El backend crea la bala desde la posición del jugador.
* La dirección de la bala sale usando aimAngle:

dirX = Math.cos(aimAngle)
dirY = Math.sin(aimAngle)

La bala debe moverse cada tick usando deltaTime:

bullet.x += bullet.dirX * bullet.speed * deltaTime;
bullet.y += bullet.dirY * bullet.speed * deltaTime;

Si la bala toca un obstáculo destructible:

* Se reduce hp del obstáculo según bullet.damage.
* Si hp <= 0, se elimina el obstáculo del mapa.
* La bala se elimina, excepto si el obstáculo es mirror.

Si la bala toca acero:

* La bala se elimina.
* El acero no recibe daño.

Si la bala toca espejo:

* La bala no se elimina.
* La bala invierte dirX y dirY.

Movimiento de jugadores:

Cada jugador debe tener:

{
id: string;
x: number;
y: number;
radius: number;
speed: number;
hp: number;
aimAngle: number;
input: PlayerInput;
lastShotAt: number;
}

El backend debe mover al jugador con moveX y moveY, pero debe normalizar el vector para que en diagonal no se mueva más rápido.

Ejemplo:

const length = Math.sqrt(moveX * moveX + moveY * moveY);

if (length > 0) {
moveX = moveX / length;
moveY = moveY / length;
}

player.x += moveX * player.speed * deltaTime;
player.y += moveY * player.speed * deltaTime;

El backend debe validar:

* moveX entre -1 y 1
* moveY entre -1 y 1
* aimAngle debe ser número válido
* shoot debe ser boolean
* no aceptar posiciones enviadas por el cliente
* no aceptar vida enviada por el cliente
* no aceptar daño enviado por el cliente

Colisiones:

Implementar colisión básica usando AABB para obstáculos rectangulares.

Para jugadores, puede usarse círculo simple con radius.

Por ahora implementar:

* bala contra obstáculos
* jugador contra bordes del mapa
* bala fuera del mapa
* jugador no debe salir del mapa

El estado enviado al frontend debe ser algo como:

{
status: 'waiting' | 'playing' | 'finished';
map: {
width: number;
height: number;
obstacles: Obstacle[];
};
players: PlayerPublicState[];
bullets: BulletPublicState[];
}

No enviar información innecesaria como inputs internos o cooldowns internos si no hace falta.

Resultado esperado:

Genera el código completo del backend con NestJS y TypeScript, separando responsabilidades por servicios.

También incluye comandos para crear el proyecto e instalar dependencias.

Usar npm.

No quiero base de datos todavía. Todo debe funcionar en memoria.
