Bush vs Decoration — key difference

  • Bush — has collision with bullets (HP depletes), can be destroyed. Used as soft cover that eventually disappears.
  • Decoration — completely ignored by all game logic. Pure visual dressing.

Mirror geometry tip

Mirrors use non-square dimensions to define the reflective face direction:

  • width > height (e.g. 140×18) → horizontal mirror, reflects bullets traveling vertically.
  • height > width (e.g. 18×130) → vertical mirror, reflects bullets traveling horizontally.

7. JSON Structure Reference

The complete GameMap JSON shape sent to clients via gameJoined:

{
  "width": 1600,
  "height": 1200,
  "obstacles": [
    {
      "id": "uuid-v4",
      "type": "rock" | "wood" | "steel" | "bush" | "decoration" | "mirror",
      "assetId": "rock_block",           // optional, falls back to type default
      "x": 360,                          // CENTER x position
      "y": 430,                          // CENTER y position
      "width": 64,                       // obstacle width  (default 64)
      "height": 64,                      // obstacle height (default 64)
      "hp": 102,                         // current HP
      "maxHp": 102,                      // max HP
      "healthRatio": 1.0,                // hp / maxHp (0.0 – 1.0)
      "destructible": true
    }
  ],
  "powerUps": [
    {
      "id": "uuid-v4",
      "type": "triple_shot" | "shotgun" | "grenade" | "laser",
      "assetId": "power_triple_shot",
      "x": 800,
      "y": 520,
      "radius": 18                       // pickup radius in pixels
    }
  ]
}

8. Prompt Template for Custom Scenarios

Paste this template into a new conversation and fill in the placeholders:

Create a Tank Arena map scenario called [SCENARIO NAME].

Map size: 1600 × 1200 px. Obstacle positions are center coordinates.

Theme: [e.g. "abandoned factory", "jungle ruins", "urban street grid", "symmetrical sniper duel"]

Layout intent:
- Number of lanes: [e.g. 3 horizontal + 2 vertical]
- Center area: [open / contested / blocked]
- Preferred obstacle density: [sparse / medium / dense]
- Symmetry: [yes (left-right) / yes (4-way) / no]

Obstacle types to include:
- bush: [count or "many" / "few" / "none"]
- wood: [count]
- rock: [count]
- steel: [count, these are permanent anchors]
- mirror: [count, specify horizontal/vertical/mixed]
- decoration: [count, visual only, no collision]

Special constraints:
- Keep the 4 corner spawn zones (within ~200 px of each corner) free of solid obstacles.
- Keep at least one clear path through the horizontal center lane (y ≈ 500–700).
- Power-up area: [keep center / spread across map / remove and relocate]

Output a JSON object with this exact structure:
{
  "obstacles": [ { "type", "x", "y", "width"?, "height"?, "assetId"? } … ],
  "powerUps": [ { "type", "x", "y" } … ]
}

Do not include "id", "hp", "maxHp", "healthRatio", or "destructible" — the server fills those from type defaults.
